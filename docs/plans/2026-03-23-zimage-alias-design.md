# Z-Image 别名预设设计

**日期：** 2026-03-23

## 目标

让下游只看到一组固定的 `z-image-*` 模型别名预设。下游调用这些别名时，中转站自动映射到上游真实模型 `zimage` 和对应分辨率，并在请求中显式传 `safe=false`。

## 已确认范围

- `/v1/models` 仅返回以下 9 个模型别名：
  - `z-image-1024x1024`
  - `z-image-1216x832`
  - `z-image-1216x688`
  - `z-image-688x1216`
  - `z-image-832x1216`
  - `z-image-1560x2048`
  - `z-image-1260x2048`
  - `z-image-2048x1260`
  - `z-image-2048x1560`
- 图片生成时，如果 `model` 命中上述别名，则忽略下游传入的 `size`，强制使用别名绑定尺寸。
- 图片请求始终显式附带 `safe=false`。

## 设计决策

### 模型预设来源

服务端维护一份静态预设映射，而不是继续把 Pollinations 上游模型列表原样透出。这样下游看到的能力边界和实际可用预设完全一致。

### 别名解析

新增一个独立模块负责：
- 定义公开的模型预设列表
- 将下游传入的别名解析成上游模型名和固定尺寸

这样可以让模型列表和生图逻辑共享同一份配置，避免前后不一致。

### 安全审核参数

当前代码没有主动传 `safe`，而上游默认值是 `false`。本次改为显式传 `safe=false`，避免上游未来默认值变化导致行为漂移。

## 非目标

- 暂不增加参考图 `image` 字段
- 暂不做后台可视化配置
- 暂不限制手工调用其他模型名
